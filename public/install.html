常见问题

三种装法有什么区别?

• 装到主屏: 最标准, 所有平台都支持, 推送最稳. Android 会自动弹安装条, iOS 需要手动点分享按钮.

• Web Clip: 仅限 iOS, 装起来快一点, 但需要你先信任一个苹果描述文件. 存储机制和 Safari 共享, 长期不用可能被 iOS 的 ITP 清理掉密钥.

• 部署你自己的: 你拿到自己的域名 + 代码完全控制. 相当于 "开一家自己的聊天室". 其他两种是"使用我们提供的聊天室".

我的聊天内容安全吗?

DAO Message 是端到端加密的 — 只有聊天双方能看到明文, 连我们服务器也看不到.

无论用哪种装法 (官方站 / Web Clip / 部署自己的), 加密都是客户端完成, 服务器只转发密文.

详细架构见 doc.daomessage.com.

部署到自己的 Cloudflare 花钱吗?

个人使用完全免费. Cloudflare Pages 免费额度:

  • 500 次构建/月 (足够个人)
  • 无限带宽
  • 免费 SSL 证书, 自定义域名

Vercel / Netlify 各自 100GB 带宽/月, 也够个人用.

Web Clip 会不会自动给我装什么奇怪的东西?

不会. Web Clip 描述文件只含一件事: 把一个 URL (chat.daomessage.com) 放到主屏图标. 无法装 App, 无法访问通讯录, 无法读短信.

你可以打开描述文件 XML 亲自看. 装完后在"设置 → VPN 与设备管理"可随时卸载.

为什么 iOS 显示"未签名 / 未验证"橙色字?

因为我们的描述文件**确实没有签名** — 这是有意的选择,不是 bug.

给 .mobileconfig 签名需要一种特殊的 S/MIME 证书 (EKU 含 emailProtection),普通 HTTPS 证书 (Cloudflare / Let's Encrypt 免费的那种) EKU 是 serverAuth,不能签. iOS 会拒绝,仍然显示红字.

能签的证书 ($12-20/年) 拿到后唯一的视觉收益是:橙色"未签名"变成"已验证 by [我们的名字]". 装出来的 Web Clip 完全一样 — 同一个 URL、同一个图标、同一个推送、同一个端到端加密.

我们选择把这 $12 省掉,因为:

  • 橙色"未签名" ≠ 不安全. 它的字面意思是"这个文件没有数字签名告诉你它来自谁"
  • 你已经从 chat.daomessage.com 这个 HTTPS 站点下载,Cloudflare 证书已经证明了来源
  • 装出来的图标只能打开一个 URL,无法获取系统权限

如果你介意,Step 2 可以用 Step 1 (Safari 分享 → 添加到主屏幕) 替代,效果完全一样.

我自部署的版本能不能签?

能. 我们的 relay 代码支持签名 — 配置 WEBCLIP_SIGNING_CERT_PATH 等三个环境变量指向你的 PEM 证书,启动时自动启用. 没配 = 走未签名路径.

详见仓库 docs/WEBCLIP_CERT_SETUP.md — 包含证书选购、转换脚本、AWS 部署一条龙.

装完怎么卸载?

• 主屏图标 (装到主屏 / Web Clip): 长按图标 → 删除

• Web Clip 描述文件: 设置 → 通用 → VPN 与设备管理 → 选中描述文件 → 移除

• 部署的站: 到对应平台 (CF / Vercel / Netlify) 删除项目

我部署的站怎么保持和官方版本同步?

你的 fork 仓库在 GitHub 上会显示 "This branch is N commits behind daomessage:main".

点击 Sync fork → Update branch, Cloudflare / Vercel / Netlify 会自动重新构建部署.

整个过程不到 1 分钟.
